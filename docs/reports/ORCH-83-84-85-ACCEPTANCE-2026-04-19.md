## ORCH-002 `#84` Orchestration Scheduler

Status: met

- Coordinator-managed orchestration state persists in relationship state as `orchestration_state`.
- `POST /tasks/{id}/orchestration` evaluates planner output, dispatches only ready work, persists fallback details, and returns the updated run model.
- Scheduler routing is feature-flagged through `FF_ORCHESTRATION_SCHEDULER`.
- Scheduler metrics are implemented in the runtime and match the monitoring artifacts:
  - request count
  - error count
  - latency
  - dispatch count
  - fallback count
  - duplicate-skip count
- Mutation authz is explicitly restricted to coordinator-capable roles in the HTTP layer.
- Supporting docs exist:
