# Milestone E — Hosted Deploy Automation (P3.2–P3.3)

Replay **phase 6 deploy closeout** against a hosted engineering-team API and operator URL. Validates GP-022 auto-merge evidence, GP-026 SRE agent gate, and GP-023 validation on promotion paths.

## Prerequisites

- Milestone D passed (`npm run milestone-d:verify`).
- Factory delivery evidence JSON from a prior A-D run (`observability/milestone-d-staging/factory-delivery/*.json`).
- Hosted API base URL and JWT secret (staging or production).
- `GITHUB_TOKEN` or `GH_TOKEN` with permission to merge the target PR.
- A passing `real-delivery-candidate-proof.v1` artifact generated by `npm run autonomy:verify-real-delivery-candidate` with final release proof, deployment-health verification, production-safety evidence, rollback evidence, GitHub API-sourced checks, branch-protection evidence, and Merge readiness. Placeholder JSON such as `{}` is rejected by preflight.
- Actual PR identity, branch, implementation commit, deployment URL, rollback target, GitHub branch-protection required checks, and passing GitHub `Merge readiness` evidence. The real-evidence gates reject simulated auto-merge, default pilot PR #271, skipped validation, SRE waivers, local/private deploy URLs, and missing candidate proof.

## Hosted phase 6 replay

```bash
export AUTH_JWT_SECRET=<hosted-jwt-secret>
export FORGEADAPTER_BASE_URL=<hosted-forgeadapter-url>  # when forge bridge required
export GITHUB_TOKEN=<github-token>
export ROLLBACK_EVIDENCE=observability/release/rollback-verification.json
export RELEASE_HEALTH_CHECK_PATH=/version
export RELEASE_BUILD_COMMAND="npm run build"
export RELEASE_COMPATIBILITY_COMMAND="npm run test:unit"
export RELEASE_VULNERABILITY_COMMAND="npm audit --audit-level=high"
export RELEASE_SECRET_COMMAND="npm run secrets:scan"
```

First generate the ordered real-delivery plan. A blocked plan exits non-zero,
marks every command with `ready: false` and `blockedBy`, and labels human output
as `[BLOCKED]`; do not run blocked commands.

```bash
npm run autonomy:plan-real-delivery -- \
  --base-url https://<hosted-api> \
  --operator-url https://<hosted-app> \
  --release-env staging \
  --repository wiinc1/engineering-team \
  --branch <branch-name> \
  --implementation-commit-sha <implementation-commit-sha> \
  --pr-url https://github.com/wiinc1/engineering-team/pull/<pr-number> \
  --pr-number <pr-number> \
  --deployment-url https://<hosted-app> \
  --rollback-target <last-known-good-release-or-url> \
  --rollback-evidence "$ROLLBACK_EVIDENCE" \
  --rollback-verified \
  --health-check-path "$RELEASE_HEALTH_CHECK_PATH" \
  --require-health-commit \
  --release-build-command "$RELEASE_BUILD_COMMAND" \
  --release-compatibility-command "$RELEASE_COMPATIBILITY_COMMAND" \
  --release-vulnerability-command "$RELEASE_VULNERABILITY_COMMAND" \
  --release-secret-command "$RELEASE_SECRET_COMMAND" \
  --candidate-test-command "npm run test:unit" \
  --final-verification-report observability/real-autonomous-delivery-verification-report.json \
  --json \
  --report observability/real-delivery-plan.json

npm run autonomy:execute-real-delivery-plan -- \
  --plan observability/real-delivery-plan.json \
  --stage pre-merge \
  --report observability/real-delivery-plan-execution-pre-merge.json \
  --json
```

After GitHub returns the merge commit, execute the saved post-merge plan. This
regenerates release artifacts for the merged commit, runs final delivery
verification, writes
`observability/real-autonomous-delivery-verification-report.json`, and persists
the execution result:

```bash
export MERGE_COMMIT_SHA=<github-merge-commit-sha-after-merge>

npm run autonomy:execute-real-delivery-plan -- \
  --plan observability/real-delivery-plan.json \
  --stage post-merge \
  --execute \
  --report observability/real-delivery-plan-execution-post-merge.json \
  --json

npm run autonomy:verify-real-delivery-plan-execution -- \
  --plan observability/real-delivery-plan.json \
  --pre-merge-report observability/real-delivery-plan-execution-pre-merge.json \
  --post-merge-report observability/real-delivery-plan-execution-post-merge.json \
  --final-verification-report observability/real-autonomous-delivery-verification-report.json \
  --report observability/real-delivery-plan-execution-audit.json \
  --json
```

### Flags

| Flag | Purpose |
| --- | --- |
| `--phase6-only` | Alias via replay script; skips Milestone A intake replay |
| `--operator-url` | Deployed app URL for SRE monitoring window |
| `--evidence-path` | Prior factory evidence to resume from |
| `npm run autonomy:plan-real-delivery` | Ordered real-delivery plan gate; blocked reports mark all commands as `ready: false` with `blockedBy` reasons |
| `npm run autonomy:execute-real-delivery-plan` | Saved-plan executor; dry-runs by default, persists a `real-autonomous-delivery-plan-execution.v1` report with `--report` and `planDigest`, refuses blocked or unready commands, and checks required env vars when `--execute` is used |
| `npm run autonomy:verify-real-delivery-plan-execution` | Final saved-plan execution audit; requires pre-merge and post-merge executor reports to match the same plan SHA-256, requires post-merge command results to pass, and requires the `real-autonomous-delivery-verification-report.v1` final verifier report to be passing |
| `npm run autonomy:preflight-real-delivery` | Local preflight that validates required real-evidence inputs before hosted replay |
| `--skip-validation` | Local smoke only; skips lint, test:unit, and standards:check in phase 6 |
| `--auto-merge` | Required for real-evidence phase 6; enables live GitHub merge |
| `--candidate-proof` | Required passing `real-delivery-candidate-proof.v1` from the pre-run verifier; preflight rejects placeholder, failed, non-GitHub, or health-unverified proof |
| `--branch-protection-json` / `--branch-protection-file` | Manual diagnostic proof only; use GitHub collection for hosted replay |
| `--rollback-evidence` | Required rollback verification artifact for the same release environment and rollback target |
| `--require-health-commit` | Requires post-deploy health evidence to prove the deployed commit SHA |
| `--release-*-command` | Required build, compatibility, vulnerability, and secret-scan evidence commands unless using prebuilt release artifacts |
| `--use-existing-release-artifacts` | Reuses prebuilt command-artifact JSON from `--release-artifact-dir`; the plan and preflight require build, compatibility, vulnerability, and secret-scan artifacts to exist, match the release environment and commit, and have `status: "passed"` |
| `--allow-local-hosted` | Permit local base URL for coordinated-stack smoke |

## Real-evidence exit criteria

- [ ] Hosted phase 6 replay reaches `phase6_complete` without `--allow-local-hosted`.
- [ ] `summary.gp026SreAgent` true with hosted SRE monitoring evidence.
- [ ] `summary.validationOk` true; do not use `--skip-validation`.
- [ ] GP-022 `autoMerge` evidence confirms GitHub merged the real PR and includes a non-fixture merge commit SHA.
- [ ] `npm run autonomy:verify-real-delivery` writes a passing `observability/real-autonomous-delivery-verification-report.json` with `--candidate-proof` for the same branch, commit, PR, changed files, deployment URL, and rollback target.

Proof artifact: `observability/milestone-hosted-staging/milestone-hosted-phase6-verify.json`
