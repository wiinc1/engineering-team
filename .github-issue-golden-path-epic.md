# Golden Path Autonomous Delivery Epic (GitHub issue body draft)

**Goal:** Run one GitHub issue end-to-end (supervised) from requirements → implement → QA fail → fix → QA pass → PM + Architect approve → local deploy validation → closeout. Document every manual step so we know what to automate next.

## Systems

- **GitHub** — issue intake, PR, merge
- **engineering-team** — local Postgres stack (audit API + UI + workflow)
- **forgeadapter** — local execution plane with full lifecycle e2e
- **OpenClaw / Hermes** — delegation (optional for GP-013)

## Acceptance criteria

- [ ] GP-001–GP-027 logged in `docs/reports/GOLDEN_PATH_PILOT_EVIDENCE.md`
- [ ] Local golden-path stack (`npm run dev:golden-path:up`) used for operator replay
- [ ] Intentional QA fail then retest pass
- [ ] forgeadapter local-stack exercised: start + resume or reject path + complete/cancel
- [ ] PM + Architect close review recorded in ET task history
- [ ] Local deploy validation recorded (GP-023: lint, test:unit, standards:check)
- [ ] `observability/golden-path-postgres-pilot.json` committed with step timestamps
- [ ] forgeadapter lifecycle-local-stack 6/6 pass captured in evidence

## Out of scope (v1)

- Hosted split deployments (UI and API on different platforms)
- Unattended operation without supervised evidence
- Redis for factory platform
