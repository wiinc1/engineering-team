## Golden Path Epic — Supervised Autonomous Delivery

**Goal:** Run one GitHub issue end-to-end (supervised) from requirements → implement → QA fail → fix → QA pass → PM + Architect approve → Vercel deploy → closeout. Document every manual step so we know what to automate next.

**Runbook:** `docs/runbooks/golden-path-autonomous-delivery.md`  
**Step inventory:** `observability/golden-path-manual-steps.json` (27 steps, all manual today)  
**Evidence template:** `docs/reports/GOLDEN_PATH_PILOT_EVIDENCE.md`

### Why this epic

We have three strong pieces that are not yet one pipeline:

- **engineering-team** — production workflow platform (Vercel + Supabase)
- **forgeadapter** — local execution plane with full lifecycle e2e
- **OpenClaw agents** — personas configured, not closed-loop wired

This epic produces the **spine map** before building automation (GP-002, GP-011, GP-016, GP-018, …).

### Template tier

**Chosen tier:** Simple  
**Change type:** docs/test-only marker + evidence artifacts  
**Production risk:** none (no auth/schema/data/infra changes)

### User story

As a Software Factory operator, I want one supervised golden-path delivery run fully logged, so every manual step has a named automation target and we can prioritize Phase A orchestration work.

### Acceptance criteria

#### Must have

- [ ] All GP-001–GP-027 steps executed or explicitly skipped with reason in `GOLDEN_PATH_PILOT_EVIDENCE.md`
- [ ] Intentional QA **fail** then **retest pass** recorded in ET task history
- [ ] forgeadapter local-stack exercised: start + resume or reject path + complete/cancel
- [ ] PM + Architect close review recorded in ET
- [ ] PR merged with required checks green
- [ ] Vercel production deploy recorded on task
- [ ] `observability/golden-path-pilot.json` filled with step timestamps
- [ ] Automation priority table validated against actual manual interventions

#### Should have

- [ ] OpenClaw live delegation smoke (`test:delegation:live-smoke:openclaw`) or visible blocker logged
- [ ] forgeadapter lifecycle-local-stack 6/6 pass captured in evidence

#### Out of scope (v1)

- Unattended operation
- Redis provisioning
- forgeadapter production deployment
- Generic deploy for arbitrary target repos

### Deliverable

Docs-only marker proving the loop completed:

- Golden-path section in README (one paragraph + link to runbook)
- Filled evidence report and pilot JSON

### Workflow

1. Operator creates child pilot task from template in runbook Phase 0
2. Execute phases 0–6 per runbook, logging each step
3. Classify manual actions: `routine observation` | `required approval` | `operator intervention`
4. Publish closeout; link follow-up automation issues for P0 gaps
