# Forge execution-readiness + local smoke seed (with affectsUi)

**Merge request:** [!268](http://192.168.1.116/wiinc1/engineering-team/-/merge_requests/268)

## Summary

Ships the engineering-team side of forgeadapter Phase 2 local-stack validation:

- `GET /tasks/:taskId/forge-execution-readiness` on audit-api (service token + `forge:read` JWT)
- Forge canonical task mapping via `lib/task-platform/forge-canonical-task.js`
- Local smoke seed helper at `lib/forge-local-smoke/seed-task.js` + `npm run forge:local-smoke:seed`
- Operator runbook: `docs/runbooks/forge-local-smoke.md`
- **`affectsUi` seed option** so forgeadapter lifecycle e2e can route UX delegate work realistically

forgeadapter CI currently pins `ENGINEERING_TEAM_REF` to this branch until this merges to `main`.

## Motivation

forgeadapter `main` runs Phase 2 local-stack e2e (`local-stack-smoke`, `lifecycle-local-stack`) in CI. Those tests require:

1. Native execution-readiness reads (no `:3003` bridge)
2. Programmatic seeding of execution-ready tasks in an isolated file-backend audit store
3. `options.affectsUi` on the seed helper for UX delegate/review lifecycle scenarios

Without this merge, CI must stay on a feature-branch checkout.
